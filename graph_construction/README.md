- Load and clean raw data about organizations involved in the network.
- Explore key features, distributions, and data quality issues.
- Standardize and enrich organization information for downstream analysis.

#### Notes:

Data sources and output files are located in the output/ and research_products_data/ directories.

## `04_preprocessing_research_products.ipynb`

Preprocessing and transformation of research product data, ensuring high data quality and a well-defined structure for subsequent analyses.

#### Main objectives:

- Analyze the structure and types of fields in the raw research products dataset.
- Quantify and report the percentage of missing values for each field.
- Filter out fields with excessive missing data and remove unnecessary or redundant fields.
- Transform and standardize key fields, especially those related to authors and subjects.
- Export a clean, structured dataset for downstream tasks.

#### Workflow overview:

- Field Structure Analysis: the notebook loads a sample research product and prints the structure and types of all fields, providing an overview of the data before any transformation.
- Missing Value Quantification: it calculates the percentage of null values for each field across the entire dataset, guiding the selection of fields to retain or remove.
- Field Filtering and Data Cleaning: fields with more than a specified threshold of missing values are excluded.
  - Additional fields deemed unnecessary for analysis are explicitly removed.
  - For the authors field, subfields such as name, surname, and provenance are removed from each author object, as well as from nested pid dictionaries.
  - Subject-related fields are parsed to extract and consolidate keywords, SDG (Sustainable Development Goals), and FOS (Fields of Science) into dedicated fields, ensuring these are always present (even if empty).

#### Export:
The cleaned and filtered dataset is exported in JSON array format, ready for further analysis.

#### Post-Processing Validation:
The notebook re-examines the filtered dataset, reporting the new percentage of missing values for each field and printing a sample record to verify the final structure.

### Final research product structure

- `authors`: A list of author objects, each containing only essential fields:
  - `fullName`: The complete name of the author.
  - `rank`: The author’s order in the publication.
  - `pid`: An object with persistent identifier information (e.g., ORCID), or null.
- `type`: the type of research product (always "publication" in the filtered dataset).
- `mainTitle`: the main title of the publication.
- `publicationDate`: the publication date.
- `id`: the unique identifier for the research product.
- `indicators`: an object containing citation and impact metrics, with subfields such as `citationCount`, `influence`, `popularity`, `impulse`, and their respective classification labels (https://graph.openaire.eu/docs/data-model/entities/other#indicator-1).
- `keywords`: A string of keywords, separated by semicolons.
- `SDG`: A string representing **Sustainable Development Goals**, possibly empty (https://sdgs.un.org/goals).
- `FOS`: A string representing **Fields of Science**, possibly empty (https://en.wikipedia.org/wiki/Fields_of_Science_and_Technology).
